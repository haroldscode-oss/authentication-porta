# Development Workflow Rules

## Phase 1: Visual Design & UI Creation
- Focus on HTML, CSS, and visual components only
- Create placeholder buttons and links that show navigation intent but don't implement functionality
- Use basic onClick handlers or href="#" for visual feedback only
- Build complete visual interface first before any backend logic

- Create comprehensive implementa
- Once visual design is complete, create BACKEND.md files for each page/component
- Document all required functionality, API endpoints, data flow, and business logic
- Specify authentication requirements, data validation, and security considerations
- Add proper error handling and user feedback

## Phase 3: Functional Implementation
- Remove placeholder elements and implement real functionality
- Use BACKEND.md files as implementation blueprints
- Integrate with actual authentication, databases, and external services

- GitHub Spark excels at UI/UX but has backend limitations due to security restrictions

- Visual completeness first ensures good UX design decisions
